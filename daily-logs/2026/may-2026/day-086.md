### 8️⃣ Wrote Custom S3 IAM Policy

Created a policy allowing:
- Listing S3 buckets
- Reading objects from only one bucket

Policy created:

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": "s3:ListAllMyBuckets",
      "Resource": "*"
    },
    {
      "Effect": "Allow",
      "Action": "s3:GetObject",
      "Resource": "arn:aws:s3:::my-bucket/*"
    }
  ]
}
```

Understood:
- Different actions may require different resources

Visualized:

```text
List Buckets
      ↓
All Buckets

Get Objects
      ↓
Specific Bucket Only
```

Learned:
- IAM policies support granular resource-level permissions

### 1️⃣6️⃣ Created Dynamic User S3 Policy

Built a policy allowing users to access only their own S3 folder.

Policy created:

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": [
        "s3:GetObject",
        "s3:PutObject"
      ],
      "Resource": "arn:aws:s3:::my-bucket/${aws:username}/*"
    }
  ]
}
```

Understood:
- IAM policies can dynamically isolate user data

Visualized:

```text
User
  ↓
Own Folder Only
```

Learned:
- Policy variables enable scalable multi-user architectures

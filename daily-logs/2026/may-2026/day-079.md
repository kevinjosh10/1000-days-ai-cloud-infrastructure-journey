### 1️⃣ Learned What a Subnet Is

Studied AWS subnet fundamentals.

Learned:
- A subnet is a smaller network created inside a VPC
- Subnets divide a VPC into smaller IP ranges
- Every subnet belongs to only ONE Availability Zone

Understood:
- Resources inside a subnet share the same network segment
- Subnets organize infrastructure logically

Visualized:

```text
VPC
  ↓
Subnets
  ↓
Resources
```

Learned:
- Subnets are foundational for cloud networking
- Network segmentation improves security and scalability

### 2️⃣ Learned How Subnet CIDR Allocation Works

Studied subnet CIDR notation and IP allocation.

Example subnet:

```text
10.0.1.0/24
```

Understood:
- `/24` provides 256 IP addresses
- Subnets are carved from the main VPC CIDR block

Learned subnet examples:

| CIDR | Total IPs |
|---|---|
| /24 | 256 |
| /25 | 128 |
| /26 | 64 |

Visualized:

```text
10.0.0.0/16
      ↓
10.0.1.0/24
```

Learned:
- Proper CIDR planning prevents network overlap
- Subnet sizing impacts scalability

---

### 3️⃣ Learned AWS Reserved IP Behavior

Studied AWS subnet IP reservation rules.

Learned:
- AWS reserves 5 IPs per subnet

Reserved addresses:

| Reserved IP | Purpose |
|---|---|
| First IP | Network address |
| Second IP | VPC router |
| Third IP | DNS |
| Fourth IP | Future AWS use |
| Last IP | Broadcast |

Example:

```text
10.0.1.0/24
```

Understood:
- Total IPs:
  - 256
- Usable IPs:
  - 251

Visualized:

```text
256 Total IPs
      ↓
5 Reserved by AWS
      ↓
251 Usable
```

Learned:
- AWS networking automatically reserves infrastructure IPs

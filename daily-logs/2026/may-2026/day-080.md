## ✅ What I Completed

### 1️⃣ Learned What an Internet Gateway (IGW) Is

Studied AWS Internet Gateway fundamentals.

Learned:
- An Internet Gateway connects a VPC to the public internet
- IGW acts as the bridge between AWS infrastructure and external networks
- Internet traffic enters and exits through the IGW

Understood:
- IGW is required for public internet connectivity
- Without an IGW, a VPC remains isolated from the internet

Visualized:

```text
Internet
    ↓
Internet Gateway
    ↓
VPC
```

Learned:
- AWS manages the infrastructure behind the IGW automatically
- Cloud engineers use IGWs to expose public-facing infrastructure

---

### 2️⃣ Learned Internet Gateway Architecture

Studied AWS-managed IGW behavior.

Learned:
- One IGW can be attached to one VPC at a time
- One VPC can only have one attached IGW
- Multiple public subnets can share the same IGW

Understood:
- IGWs are fully managed by AWS
- IGWs are highly available and fault tolerant

Visualized:

```text
VPC
  ↓
Single IGW
  ↓
Internet
```

Learned:
- Engineers do not manage servers behind the IGW
- AWS automatically handles scaling and redundancy

### 4️⃣ Learned What a Route Table Is

Studied AWS route table fundamentals.

Learned:
- Route tables determine where network traffic goes
- Every subnet must be associated with a route table
- Routes define packet forwarding behavior

Understood:
- Route tables act like traffic maps
- AWS networking depends heavily on route configuration

Visualized:

```text
Traffic
   ↓
Route Table
   ↓
Destination
```

Learned:
- Incorrect routing breaks connectivity
- Cloud engineers use route tables to control network isolation

### 5️⃣ Learned Public Route Table Architecture

Studied internet-facing route table configuration.

Created public route table:

```text
public-rt
```

Added route:

```text
0.0.0.0/0 → Internet Gateway
```

Understood:
- `0.0.0.0/0` means all IPv4 destinations
- This is called the default route

Visualized:

```text
Any Destination
       ↓
0.0.0.0/0
       ↓
Internet Gateway
```

Learned:
- Public route tables allow internet-bound traffic
- Public subnets require IGW routes

Understood:

Learned:
- Internal VPC communication works automatically
- AWS manages local routing internally

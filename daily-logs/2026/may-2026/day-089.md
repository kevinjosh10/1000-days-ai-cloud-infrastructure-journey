## ✅ What I Completed

### 1️⃣ Learned About Permission Boundaries

Studied AWS Permission Boundaries and how they act as maximum permission limits for IAM users and roles.

Understood:
- Permission boundaries limit maximum permissions
- They do not grant permissions themselves
- Effective permissions are the intersection of identity policy and boundary policy

Visualized:

```text
Identity Policy
       ∩
Permission Boundary
       ↓
Effective Permissions
```

Learned:
- Permission boundaries create safe IAM guardrails

---

### 2️⃣ Learned Effective Permissions Intersection Logic

Studied how AWS calculates final permissions when boundaries are attached.

Understood:
- Both identity policy and boundary must allow access
- Missing permission in either policy results in denial

Visualized:

```text
Allow in Identity Policy
           +
Allow in Boundary
           ↓
Final Allow
```

Learned:
- AWS authorization relies on layered permission evaluation

---

### 3️⃣ Learned Why Permission Boundaries Are Important

Studied enterprise use cases for permission boundaries.

Understood:
- Organizations delegate IAM management safely
- Developers can create roles without exceeding allowed limits
- Boundaries prevent privilege escalation

Visualized:

```text
Developer Creates Role
          ↓
Boundary Limits Maximum Access
```

Learned:
- Permission boundaries enable secure delegated administration

### 4️⃣ Created DevBoundary Policy

Created a permission boundary policy named `DevBoundary`.

Policy created:

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": [
        "ec2:*",
        "s3:*",
        "cloudwatch:*"
      ],
      "Resource": "*"
    }
  ]
}
```

Visualized:

```text
Allowed Services
      ↓
EC2 + S3 + CloudWatch Only
```

Learned:
- Permission boundaries define maximum allowed service scope

### 1️⃣3️⃣ Compared Permission Boundaries vs SCPs

Studied differences between boundaries and SCPs.

Understood:
- Boundaries apply to IAM entities
- SCPs apply to entire AWS accounts/OUs
- SCPs affect root users
- Boundaries do not affect root users

Visualized:


- AWS provides multiple layers of governance

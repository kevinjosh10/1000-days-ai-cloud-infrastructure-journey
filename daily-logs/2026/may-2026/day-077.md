### 3️⃣ Built the Application Tier Using Flask

Created a private application server using EC2 and Python Flask.

Installed Python dependencies:

```bash
sudo apt install python3-pip -y
pip3 install flask
```

Created Flask application:

```python
from flask import Flask

app = Flask(__name__)

@app.route('/')
def home():
    return "Hello from App Tier"

if __name__ == '__main__':
    app.run(host='0.0.0.0', port=5000)
```

Started Flask app:

```bash
python3 app.py
```

Verified locally:

```bash

```

Learned:

- Application tier should not be publicly exposed

Understood:
- Backend services communicate internally
- Internal APIs power cloud applications

### 4️⃣ Learned Reverse Proxy Architecture Using nginx

Configured nginx to forward requests to the Flask application.

Edited nginx configuration:

```bash
sudo nano /etc/nginx/sites-available/default
```

Configured reverse proxy:

```nginx
server {
    listen 80;

    location / {
        proxy_pass http://APP_PRIVATE_IP:5000;
    }
}
```

Restarted nginx:

```bash
sudo systemctl restart nginx
```

Understood request flow:

```text
User
  ↓
nginx
  ↓
Flask App
```

Learned:
- Reverse proxies protect backend applications
- Backend servers remain hidden from the internet
- nginx forwards traffic internally

Understood:
- Reverse proxy architecture is foundational in cloud engineering
- Kubernetes ingress controllers work similarly

### 5️⃣ Practiced Multi-Tier Security Group Design

Created production-style Security Groups.

Security Groups created:
- web-sg
- app-sg
- db-sg

Configured rules:

#### web-sg

Inbound:
- HTTP (80) from:
  - 0.0.0.0/0
- SSH (22) from:
  - My IP

#### app-sg

Inbound:
- Port 5000 from:
  - web-sg

#### db-sg

Inbound:
- Database port from:
  - app-sg

Learned:
- Security Groups can reference other Security Groups
- Infrastructure trust should be SG-based, not IP-based

Visualized:

```text
web-sg
   ↓
app-sg
   ↓
db-sg
```

Understood:
- SG-to-SG communication is production best practice
- Dynamic infrastructure benefits from security abstraction

### 6️⃣ Learned Why SG Referencing Is Superior to IP Rules

Studied enterprise networking practices.

Compared approaches:

Understood:

Learned:

### 9️⃣ Learned Infrastructure Automation Using boto3

Began automating infrastructure deployment using Python boto3.

Installed boto3:

```bash
pip install boto3
```

Configured AWS CLI:

```bash
aws configure
```

Created starter deployment script:

```python
import boto3

ec2 = boto3.resource('ec2')

instances = ec2.create_instances(
    ImageId='ami-xxxxxxxx',
    MinCount=1,
    MaxCount=1,
    InstanceType='t2.micro',
    KeyName='your-keypair',
)

print("Instance launched")
```

Learned:
- boto3 enables Infrastructure as Code using Python
- AWS resources can be provisioned programmatically

Understood:
- Infrastructure automation is essential in DevOps
- Manual infrastructure management does not scale

Visualized:

```text
Python Script
      ↓
AWS API
      ↓
Infrastructure Created
```

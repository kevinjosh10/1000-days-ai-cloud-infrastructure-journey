### 1️⃣ Learned What a VPC Endpoint Is

Studied AWS VPC Endpoint fundamentals.

Learned:
- VPC Endpoints allow private access to AWS services
- Traffic stays inside AWS network
- No Internet Gateway required
- No NAT Gateway required

Understood:
- Endpoints improve security and reduce NAT costs
- Private subnets can directly communicate with AWS services

Visualized:

```text

      ↓
VPC Endpoint
      ↓

```

Learned:
- AWS internal backbone network handles endpoint traffic privately

### 2️⃣ Understood Why VPC Endpoints Exist

Studied traditional private subnet internet flow.

Normal architecture:

```text
Private EC2
     ↓
NAT Gateway
     ↓
Internet Gateway
     ↓
S3
```

Understood problems:
- NAT Gateway costs money
- Traffic touches internet path
- Increased attack surface
- More latency

Visualized:

```text
Private Resource
      ↓
Unnecessary Internet Path
```

Learned:
- VPC Endpoints eliminate unnecessary internet routing

### 9️⃣ Compared Gateway vs Interface Endpoints

Studied architectural differences.

| Feature | Gateway Endpoint | Interface Endpoint |
|---|---|---|
| Services | S3, DynamoDB | Most AWS Services |
| Uses Route Tables | Yes | No |
| Uses ENI | No | Yes |
| Cost | Free | Usually Paid |
| Powered By | Route Entries | AWS PrivateLink |

Understood:
- Different endpoint types solve different networking problems

Visualized:

```text
Gateway Endpoint → Route Based
Interface Endpoint → ENI Based
```

Learned:
- Endpoint architecture depends on AWS service design

## 🔄 CloudFront Cache Invalidations

Implemented automatic invalidations after deployment.

Command:

```bash
aws cloudfront create-invalidation \
--distribution-id DISTRIBUTION_ID \
--paths "/*"
```

Benefits:

- Immediate content updates
- Eliminates stale cache
- Global refresh of website assets

## 🚀 End User Workflow

### Step 1

User visits website.

### Step 2

DNS routes request to nearest CloudFront Edge Location.

### Step 3

CloudFront Function executes.

### Step 4

Cache lookup performed.

#### Cache Hit

```text
Served directly from Edge Location
```

#### Cache Miss

```text
Edge Location
       ↓
S3 Origin
       ↓
Cache Response
       ↓
Serve User
```

### Step 5

Browser renders website assets.

Result:

- Fast loading
- Secure HTTPS delivery
- Low latency experience

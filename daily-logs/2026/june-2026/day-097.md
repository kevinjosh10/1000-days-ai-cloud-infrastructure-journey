# 🌐 What is Route 53?

Amazon Route 53 is AWS's scalable Domain Name System (DNS) service.

Purpose:

```text
Domain Name
      ↓
IP Address
```

Example:

```text
www.example.com
      ↓
54.210.100.50
```

Route 53 allows users to access applications using human-readable domain names instead of IP addresses.

---

# Why is it Called Route 53?

The name comes from:

```text
Port 53
```

The standard port used by DNS services.

---

# 🔍 DNS Resolution Process

When a user visits:

```text
www.example.com
```

The DNS lookup process occurs:

```text
User Browser
      ↓
Recursive Resolver
      ↓
Root DNS Server
      ↓
TLD Server (.com)
      ↓
Route 53 Hosted Zone
      ↓
Returns IP Address
      ↓
Application Server
```

Example:

```text
www.example.com
      ↓
54.210.100.50
```

Route 53 consists of:

---

# 📁 Hosted Zones

A Hosted Zone stores DNS records for a domain.

Example:

```text
example.com
```

Contains:

```text
www.example.com
api.example.com
mail.example.com
```

---

## Public Hosted Zone

Accessible from the internet.

Example:

```text
example.com
```

Used for:

- Websites
- APIs
- Public applications

Anyone on the internet can resolve the records.

---

## Private Hosted Zone

Accessible only inside associated VPCs.

Example:

```text
db.internal.local
```

Used for:

- Databases
- Internal applications
- Microservices

Only resources within the VPC can resolve these names.

---

# 📝 DNS Record Types

DNS records determine where traffic is routed.

---

## A Record

Maps a domain to an IPv4 address.

Example:

```text
app.example.com
      ↓
54.210.10.15
```

---

## AAAA Record

Maps a domain to an IPv6 address.

Example:

```text
app.example.com
      ↓
2600:1f18::1234
```

---

## CNAME Record

Creates an alias.

Example:

```text
www.example.com
      ↓
app.example.com
```

---

## MX Record

Mail Exchange record.

Used for email routing.

Example:

```text
example.com
      ↓
mail server
```

---

## TXT Record

Stores text information.

Common uses:

- Domain verification
- SPF
- DKIM
- DMARC

Example:

```text
google-site-verification=abc123
```

---

# 🔄 DNS Propagation

When DNS records change:

```text
Old Record
      ↓
New Record
```

The changes must propagate across DNS servers worldwide.

This process is called:

```text
DNS Propagation
```

Propagation can take:

```text
Seconds to Hours
```

depending on TTL values.

---

# ⏱️ TTL (Time To Live)

TTL controls how long DNS resolvers cache records.

Example:

```text
300 seconds
```

Meaning:

```text
DNS cache remains valid for 5 minutes
```

Lower TTL:

```text
Faster updates
More DNS queries
```

Higher TTL:

```text
Slower updates
Lower DNS query volume
```

# 🚦 Route 53 Routing Policies

Routing policies determine how DNS responses are returned to users.

---

# 1️⃣ Simple Routing

Default routing policy.

```text
User
 ↓
Single Resource
```

Example:

```text
www.example.com
      ↓
EC2 Instance
```

Best for:

- Single applications
- Basic DNS configurations

---

# 2️⃣ Weighted Routing

Distributes traffic based on assigned weights.

Example:

```text
us-east-1 = 90
eu-west-1 = 10
```

Traffic distribution:

```text
90% → us-east-1
10% → eu-west-1
```

---

## Use Cases

### A/B Testing

```text
Version A = 90%
Version B = 10%
```

Measure performance before full deployment.

---

### Blue-Green Deployment

```text
Blue = Current Version
Green = New Version
```

Traffic shift:

```text
100% → Blue
90%  → Blue
10%  → Green
```

Gradually move traffic to the new version.

---

# 3️⃣ Latency-Based Routing

Routes users to the region with the lowest network latency.

Example:

```text
India User
      ↓
Mumbai Region
```

```text
US User
      ↓
Virginia Region
```

Benefits:

- Faster response times
- Better user experience

---

# 4️⃣ Failover Routing

Used for Disaster Recovery.

Architecture:

```text
Primary Resource
       ↓
Health Check
       ↓
Secondary Resource
```

---

## Normal Operation

```text
Primary = Healthy
```

Traffic:

```text
100% → Primary
```

---

## Failure Scenario

```text
Primary = Unhealthy
```

Route 53 automatically routes traffic to:

```text
Secondary Resource
```

Example:

```text
Primary = EC2
Secondary = S3 Static Website
```

---

## Active-Passive Disaster Recovery

```text
Primary → Active
Backup  → Passive
```

Backup remains idle until the primary resource fails.

---

# 5️⃣ Geolocation Routing

Routes traffic based on user location.

Example:

```text
India    → Indian Website
USA      → US Website
Germany  → EU Website
```

Common use cases:

- Language localization
- Regional compliance
- Country-specific content

---

# 6️⃣ Geoproximity Routing

Routes users based on geographic distance.

Example:

```text
Mumbai Region
Singapore Region
```

Users are routed to the nearest endpoint.

Supports:

```text
Traffic Bias
```

to influence traffic distribution between regions.

---

# 7️⃣ Multivalue Answer Routing

Returns multiple healthy IP addresses.

Example:

```text
10.0.0.1
10.0.0.2
10.0.0.3
```

If a server becomes unhealthy:

```text
10.0.0.2
```

is removed from DNS responses.

Benefits:

- Basic load balancing
- Improved availability
- Health-aware DNS responses

### 5️⃣ Learned Primary Keys

Studied how DynamoDB uniquely identifies data.

Understood:

- Primary keys must uniquely identify items
- Two primary key options exist

Visualized:

```text

      ↓

```

Learned:
- Primary key design is the most important DynamoDB modeling decision

### 6️⃣ Studied Partition Keys

Learned how DynamoDB distributes data.

Understood:
- Partition key determines storage location
- Data is distributed using hashing
- Good partition keys improve scalability

Visualized:

```text
userId
   ↓
Hash
   ↓
Partition
```

Learned:
- Evenly distributed partition keys prevent hot partitions

---

### 7️⃣ Learned Composite Primary Keys

Studied partition key plus sort key design.

Understood:
- Partition Key + Sort Key
- Supports one-to-many relationships
- Enables efficient querying

Visualized:

```text

   +
sessionId
```

Learned:
- Composite keys are widely used in real-world DynamoDB applications

### 8️⃣ Studied Sort Keys

Understood:

### 1️⃣6️⃣ Learned Eventually Consistent Reads

Studied DynamoDB's default consistency model.

Understood:
- Lower cost
- Faster performance
- Reads may briefly return stale data

Visualized:

```text
Write
  ↓
Read
  ↓
Possibly Older Value
```

Learned:
- Eventual consistency is sufficient for many applications

---

### 1️⃣7️⃣ Studied Strongly Consistent Reads

Learned guaranteed consistency behavior.

Understood:
- Always returns latest write
- Higher read cost
- Greater accuracy

Visualized:

```text
Write
  ↓
Read
  ↓
Latest Value
```

Learned:
- Strong consistency is required for critical workloads

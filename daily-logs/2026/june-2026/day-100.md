## ✅ What I Completed

### 1️⃣ Learned About RDS Multi-AZ Deployments

Studied how AWS provides high availability for production databases.

Understood:
- A standby database is created in a different Availability Zone
- Data is replicated synchronously
- Standby remains ready for failover

Visualized:

```text
Primary DB (AZ-A)
        ↓
Synchronous Replication
        ↓
Standby DB (AZ-B)
```

Learned:
- Multi-AZ is designed for high availability, not performance improvement

---

### 2️⃣ Studied Synchronous Replication

Learned how data consistency is maintained.

Understood:
- Every write is copied to the standby database
- Primary waits for standby confirmation
- Data remains synchronized

Visualized:

```text
Write Request
      ↓
Primary DB
      ↓
Standby DB Confirmation
      ↓
Write Completed
```

Learned:
- Synchronous replication minimizes data loss during failures

---

### 3️⃣ Learned About Automatic Failover

Studied what happens during database failures.

Understood:
- AWS automatically switches to the standby
- Failover usually occurs within 1–2 minutes
- Applications continue using the same endpoint

Visualized:

```text
Primary Failure
       ↓
Automatic Failover
       ↓
Standby Becomes Primary
```

Learned:
- Multi-AZ improves application resilience without manual intervention

---

### 4️⃣ Verified Multi-AZ Standby Limitations

Learned an important AWS exam concept.

Understood:
- Standby databases cannot serve read traffic
- Standby databases cannot be queried
- Their only purpose is failover

Visualized:

```text
Primary DB
     ↓
Standby DB
(Not Readable)
```

Learned:
- Multi-AZ does not increase database performance

### 7️⃣ Studied Asynchronous Replication

Learned how replication differs from Multi-AZ.

Understood:
- Changes are copied after commits occur
- Replication lag may exist
- Performance impact is lower

Visualized:

```text

      ↓

Replica Updated
```

Learned:
- Read Replicas prioritize scalability over availability

### 1️⃣5️⃣ Studied Point-in-Time Recovery (PITR)

Learned one of the most important backup features.

Understood:

- Accidental deletions can be reversed
- Database corruption can be corrected

Visualized:

Learned:
- PITR provides extremely granular recovery

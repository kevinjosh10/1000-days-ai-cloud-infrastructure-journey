# 🗄️ What is Amazon RDS?

Amazon Relational Database Service (RDS) is a managed database service that allows you to run relational databases without managing the underlying infrastructure.

Instead of manually managing servers:

```text
Install Database
Patch OS
Configure Backups
Monitor Performance
Handle Failures
```

AWS automates these operational tasks.

---

# Why Use RDS?

Traditional database administration requires:

```text
Server Management
Database Installation
Backups
Patching
Monitoring
High Availability Setup
```

With RDS:

```text
AWS Manages Infrastructure
You Manage Data
```

This significantly reduces operational overhead.

```text

Scalability
Durability
```

## Aurora Performance

According to AWS:

```text

```

## Aurora Storage Architecture

Aurora automatically stores:

```text

```

Distributed across:

```text

```

Architecture:

```text

```

```text

Automatic Recovery
Fault Tolerance
```

# 🔒 RDS Deployment Best Practices

Databases should not be directly accessible from the internet.

Recommended architecture:

```text
Internet
    ↓
Public Subnet
    ↓
EC2 Bastion Host
    ↓
Private Subnet
    ↓
RDS Database
```

---

## Public Accessibility

Production databases should be configured as:

```text
Publicly Accessible = No
```

Reason:

```text
Improved Security
Reduced Attack Surface
```

This is a common AWS exam question.

# 🔗 Connecting to RDS

AWS provides a database endpoint.

Example:

```text
mydb.abc123.us-east-1.rds.amazonaws.com
```

Connect from an EC2 instance:

```bash
mysql -h mydb.abc123.us-east-1.rds.amazonaws.com -u admin -p
```

Verify connection:

```sql
SHOW DATABASES;
```

Example output:

```sql
mysql
information_schema
performance_schema
```

Successful output confirms connectivity.

# 🛡️ RDS Security Groups

Security Groups control database access.

Recommended design:

```text
EC2 Security Group
        ↓
RDS Security Group
```

Allow:

```text
Port 3306
Source = EC2 Security Group
```

Avoid:

```text
0.0.0.0/0
```

for production databases.

# ⚙️ RDS Parameter Groups

Parameter Groups control database engine settings.

Examples:

```text
max_connections
autocommit
query_cache_size
innodb_buffer_pool_size
```

Instead of editing database configuration files directly, changes are managed through Parameter Groups.

---

## Example

Increase maximum client connections:

```text
max_connections = 500
```

Apply the Parameter Group and the database uses the new configuration.

---

## Exam Shortcut

```text
Parameter Group = Database Settings
```

---

# ➕ RDS Option Groups

Option Groups provide additional database engine features.

Examples:

```text
Oracle APEX
Oracle TDE
SQL Server Features
```

Think of Option Groups as feature extensions.

---

## Exam Shortcut

```text
Parameter Group = Settings

Option Group = Features
```

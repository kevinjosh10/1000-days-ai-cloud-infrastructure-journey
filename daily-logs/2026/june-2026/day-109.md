# 🌐 What is API Gateway?

Amazon API Gateway is a fully managed AWS service that allows developers to create, publish, secure, monitor, and maintain APIs at any scale.

Purpose:

```text
Client Request
      ↓
API Gateway
      ↓
Backend Service
```

Example:

```text
User
  ↓
API Gateway
  ↓
Lambda Function
  ↓
DynamoDB
```

API Gateway acts as the front door for applications and microservices.

---

# Why Use API Gateway?

Without API Gateway:

```text
Client
   ↓
Direct Access
   ↓
Backend
```

Problems:

- Security management
- Request validation
- Rate limiting
- Monitoring
- Scalability

With API Gateway:

```text
Client
   ↓
API Gateway
   ↓
Backend
```

Benefits:

- Authentication
- Authorization
- Monitoring
- Throttling
- Request validation
- Automatic scaling

# 🏗️ API Gateway Types

AWS provides three API types.

---

# 1️⃣ REST API

The most feature-rich API type.

Supports:

```text
API Keys
Usage Plans
Caching
Request Validation
Mapping Templates
Authorization
```

Architecture:

```text
Client
   ↓
REST API
   ↓
Lambda
```

Best for:

- Enterprise APIs
- Complex integrations
- Advanced API management

---

# 2️⃣ HTTP API

A newer, lightweight API type.

Benefits:

```text
Lower Cost
Lower Latency
Simpler Configuration
```

Architecture:

```text
Client
   ↓
HTTP API
   ↓
Lambda
```

Best for:

- Serverless applications
- Microservices
- Cost-sensitive workloads

---

# 3️⃣ WebSocket API

Provides real-time bidirectional communication.

Architecture:

```text
Client
   ↕
WebSocket API
   ↕
Backend
```

Use Cases:

```text
Chat Applications
Gaming
Live Notifications
Stock Tickers
```

---

# ⚖️ API Type Comparison

| Feature | REST API | HTTP API | WebSocket API |
|----------|----------|----------|----------|
| Cost | Higher | Lower | Medium |
| Features | Most | Limited | Real-Time |
| Latency | Higher | Lower | Low |
| Bidirectional | No | No | Yes |

# 🚀 API Deployments & Stages

Stages represent environments.

Common stages:

```text
dev
staging
prod
```

Each stage gets its own endpoint.

Example:

```text
https://abc.execute-api.us-east-1.amazonaws.com/dev

https://abc.execute-api.us-east-1.amazonaws.com/staging

https://abc.execute-api.us-east-1.amazonaws.com/prod
```

---

# Why Use Stages?

Development:

```text
Developers
     ↓
DEV Stage
```

Testing:

```text
QA Team
     ↓
STAGING Stage
```

Production:

```text
Customers
     ↓
PROD Stage
```

This enables safe deployments and testing.

---

# 🔄 Lambda Versions & Aliases

Example:

```text
DEV     → Lambda Alias Dev

PROD    → Lambda Alias Prod
```

Benefits:

- Environment isolation
- Controlled deployments
- Easier rollbacks

# 🛡️ Request Validation

API Gateway can validate requests before they reach Lambda.

Example Request:

```json
{
  "name": "Kevin",
  "email": "kevin@example.com"
}
```

Required Fields:

```text
name
email
```

Invalid requests are rejected automatically.

Benefits:

```text
Reduce Lambda Invocations
Lower Costs
Better Security
```

---

# 🔄 Mapping Templates

Mapping templates transform requests and responses.

Incoming Request:

```json
{
  "username": "Kevin"
}
```

Transform To:

```json
{
  "name": "Kevin"
}
```

Flow:

```text
Client
   ↓
Mapping Template
   ↓
Lambda
```

Useful when backend formats differ from client formats.

---

# 🚦 Throttling

Throttling limits API requests.

Example Limit:

```text
100 Requests/Second
```

If:

```text
500 Requests Arrive
```

Then:

```text
100 Accepted
400 Rejected
```

Returned Response:

```http
429 Too Many Requests
```

Benefits:

- Prevent abuse
- Protect backend systems
- Control costs

# ⚡ Lambda Proxy Integration

One of the most common integration methods.

Without Proxy Integration:

```text
API Gateway
      ↓
Transforms Request
      ↓
Lambda
```

With Proxy Integration:

```text
API Gateway
      ↓
Entire Request
      ↓
Lambda
```

Lambda receives the full request.

# 📤 Lambda Response Format

Lambda must return:

```json
{
  "statusCode": 200,
  "headers": {
    "Content-Type": "application/json"
  },
  "body": "{\"message\":\"Success\"}"
}
```

API Gateway converts it into an HTTP response.

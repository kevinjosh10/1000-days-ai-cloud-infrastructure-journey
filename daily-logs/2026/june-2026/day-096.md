# 🔒 HTTPS on CloudFront

## Why HTTPS?

HTTPS encrypts data transferred between users and your application.

Benefits:

- Data encryption
- Improved security
- User trust
- Better SEO rankings
- Protection against man-in-the-middle attacks

---

## AWS Certificate Manager (ACM)

AWS Certificate Manager provides free SSL/TLS certificates.

CloudFront requires certificates to be created in:

```text
us-east-1 (N. Virginia)
```

Important:

```text
CloudFront can only use ACM certificates
created in us-east-1.
```

Even if:

```text
S3 Bucket = ap-south-1
CloudFront = Global
Certificate = us-east-1
```

---

## HTTPS Setup Workflow

```text
ACM Certificate
        ↓
CloudFront Distribution
        ↓
Custom Domain
        ↓
HTTPS Enabled Website
```

Steps:

1. Open ACM Console
2. Switch to us-east-1
3. Request certificate
4. Validate ownership via DNS
5. Attach certificate to CloudFront

# 🔄 Viewer Protocol Policy

CloudFront controls how HTTP requests are handled.

Available options:

### Allow HTTP and HTTPS

```text
HTTP  → Allowed
HTTPS → Allowed
```

Not recommended.

---

### HTTPS Only

```text
HTTP Requests → Rejected
HTTPS Requests → Allowed
```

---

### Redirect HTTP to HTTPS

Recommended option.

```text
User
 ↓
HTTP Request
 ↓
CloudFront
 ↓
301 Redirect
 ↓
HTTPS Request
 ↓
Website
```

Benefits:

- Better security
- Improved SEO
- Consistent user experience

# 🚀 CloudFront Cache Invalidations

CloudFront caches content at edge locations.

Problem:

```text
S3 Object Updated
        ↓
CloudFront Still Serves Cached Version
```

Users may continue seeing old content.

---

## Solution: Invalidation

Invalidate cached objects to force CloudFront to fetch updated files from origin.

Example:

```bash
aws cloudfront create-invalidation \
--distribution-id E123456789 \
--paths "/*"
```

Meaning:

```text
Invalidate all cached content
```

---

## Invalidate Specific Files

```bash
aws cloudfront create-invalidation \
--distribution-id E123456789 \
--paths "/index.html"
```

Only removes:

```text
/index.html
```

from cache.

---

## Pricing

CloudFront provides:

```text
First 1000 invalidation paths/month free
```

After that:

```text
Additional charges apply
```

---

## Cache Busting

Production environments often avoid invalidations by using versioned files.

Example:

```text
app-v1.js
app-v2.js
app-v3.js
```

Benefits:

- Faster deployments
- Reduced invalidation costs
- Better cache efficiency

# ⚡ CloudFront Functions

CloudFront Functions are lightweight JavaScript functions that run at edge locations.

Characteristics:

- JavaScript only
- Ultra-low latency
- Less than 1 ms execution time
- Very low cost

Supported Events:

```text
Viewer Request
Viewer Response
```

---

## Common Use Cases

- URL rewrites
- Redirects
- Header modifications
- Authentication checks
- Geo-based routing

Example:

```text
/home
 ↓
/dashboard
```

---

# 🔥 Lambda@Edge

Lambda@Edge extends CloudFront with full AWS Lambda functionality.

Characteristics:

- Node.js
- Python
- More powerful processing
- Higher cost
- Up to 5 seconds execution time

Supported Events:

```text
Viewer Request
Viewer Response
Origin Request
Origin Response
```

---

## Common Use Cases

- Dynamic content generation
- Image processing
- Personalization
- API integrations
- Advanced authentication

---

# 📊 CloudFront Functions vs Lambda@Edge

| Feature | CloudFront Functions | Lambda@Edge |
|----------|----------|----------|
| Runtime | JavaScript | Node.js / Python |
| Speed | < 1 ms | Up to 5 sec |
| Cost | Very Low | Higher |
| Viewer Request | Yes | Yes |
| Viewer Response | Yes | Yes |
| Origin Request | No | Yes |
| Origin Response | No | Yes |
| Complexity | Simple | Advanced |

---

## Rule of Thumb

Use:

```text
CloudFront Functions
```

for:

- Redirects
- URL rewrites
- Header manipulation

Use:

```text
Lambda@Edge
```

for:

- Complex business logic
- Dynamic responses
- Origin modifications

# 🛡️ AWS WAF (Web Application Firewall)

AWS WAF protects web applications from common web attacks.

Traffic Flow:

```text
Internet
    ↓
AWS WAF
    ↓
CloudFront
    ↓
Origin
```

Malicious requests are blocked before reaching your application.

---

# Web ACL

Web ACL stands for:

```text
Web Access Control List
```

A Web ACL contains security rules that inspect incoming requests.

Examples:

- Allow traffic
- Block traffic
- Rate limit requests
- Filter malicious payloads

---

# AWS Managed Rule Groups

AWS provides preconfigured security rules.

Common managed rule group:

```text
AWSManagedRulesCommonRuleSet
```

Provides protection against:

- SQL Injection
- Cross-Site Scripting (XSS)
- Known malicious patterns
- Common web exploits

Benefits:

- Easy deployment
- AWS maintained
- Automatic updates

---

# OWASP Top 10 Protection

OWASP Top 10 represents the most critical web application security risks.

Examples:

### SQL Injection

```sql
' OR 1=1 --
```

Attempts to manipulate database queries.

---

### Cross-Site Scripting (XSS)

```html
<script>
malicious_code()
</script>
```

Attempts to execute malicious scripts.

---

### Broken Access Control

Users gain access to resources they should not access.

---

### Security Misconfiguration

Improperly configured systems expose vulnerabilities.

---

AWS WAF helps mitigate many of these threats before requests reach your application.
